# 资料查阅与问题解决记录（Research Log）

> 用于课程评分项“文档查阅与问题解决能力”。这里记录关键资料点与对应落地结论（不追求长篇，追求可追溯）。

## 1) KeePassXC 模块拆解与对标
- 对标点：分组/标签、安全报告、互操作（CSV）、网络能力（favicon/泄露检查）。
- 结论：课程项目优先实现“可演示闭环”，从 CSV 导入、分组树与安全报告开始，浏览器扩展对接后置。

## 2) Qt + SQLite 多线程访问
- 关键点：`QSqlDatabase` 连接不能跨线程共享；Worker 线程必须创建自己的连接（连接名唯一）。
- 落地：安全报告扫描与 CSV 导入均使用独立连接，并在结束后显式关闭/移除连接，避免告警与潜在崩溃。

## 3) Pwned Passwords（k-anonymity 思路）
- 关键点：只上传 SHA-1 的前 5 位前缀，服务器返回后缀列表与次数；本地匹配后缀判断是否泄露。
- 落地：安全报告提供“在线泄露检查（需要联网）”开关；查询结果按前缀缓存到 SQLite，减少网络开销并支持离线复现测试。

## 4) favicon 获取与缓存策略
- 关键点：最小可行路径为 `https://<host>/favicon.ico`；必须做缓存与失败降级，避免阻塞 UI/影响体验。
- 落地：列表按 URL host 拉取 favicon，落地缓存表并设置 TTL；离线时展示已缓存图标。
