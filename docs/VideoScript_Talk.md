## 2) 0:20–1:20 架构说明（分层 + 数据落盘）
【操作】
- 口头讲解即可；需要画面时可打开 `docs/Framework.md` 或报告中的架构图/ER 图。

【台词】
- “整体按分层实现：UI 页面层（`src/pages`）负责交互；业务层（`src/password`）负责 Vault/扫描/匹配；持久化层用 QtSql 操作 SQLite。”
- “数据落盘使用 `QStandardPaths::AppDataLocation`：密码库是本地 SQLite 文件，日志写到 `toolbox.log`，便于验收复现。”
- “安全原则：主密码不明文保存，敏感字段（密码/备注）以加密后的 BLOB 写入数据库；Vault 锁定时限制敏感操作和联网行为。”

## 5) 3:30–4:10 搜索/筛选（Model/View 解耦展示）
【操作】
1) 在搜索框输入 `github` 或 `demo`，观察表格实时过滤。
2) 清空搜索恢复。

【台词】
- “筛选通过 proxy model 实现：UI 只更新筛选条件，模型层负责过滤，避免频繁拼 SQL，交互响应更稳定。”

## 6) 4:10–4:50 复制保护 + 密码生成器（安全体验点）
【操作】
1) 选中条目 → 点底部 `复制密码` → 弹窗确认 → 粘贴到记事本验证。
2) 点 `编辑` → 在密码框点击 `生成` → 调整规则 → 使用并保存。

【台词】
- “复制密码有二次确认，并在 15 秒后自动清空剪贴板，降低泄露风险。”
- “生成器支持自定义长度/字符集，并实时显示强度评分，方便演示交互与安全提示。”

## 7) 4:50–6:00 安全报告（多线程 + 可选联网检查）
【操作】
1) 点击顶部 `安全报告`，展示扫描中 UI 不冻结。
2) 扫描完成后，展示弱密码/重复密码结果；双击某行定位回条目编辑（可选）。
3) 若网络稳定：勾选 `在线泄露检查（需要联网）` → 点 `重新扫描`（可选加分）。

【台词】
- “安全报告扫描放在 worker 线程执行，通过信号把进度与结果回传 UI，避免主线程卡顿。”
- “在线泄露检查使用匿名哈希前缀查询（k-anonymity 思路），不会上传明文密码；并把响应按前缀缓存到 SQLite，减少重复请求。”

## 8) 6:00–6:40 网络能力：favicon 拉取与缓存
【操作】
- 回到主界面：让带 URL 的条目显示网站图标（首次可能稍等 1–2 秒）。

【台词】
- “网站图标通过 `QNetworkAccessManager` 拉取并缓存到 SQLite；Vault 锁定时禁用联网，保证锁定态不发请求。”

## 9) 6:40–7:30 文件能力：加密备份导入/导出（.tbxpm）
【操作】
1) 点击 `导出备份` → 选择保存路径 → 输入 `DemoBackup123!`（两次确认）→ 完成提示。
2) 时间允许可演示 `导入备份`：选择刚导出的文件 → 输入备份密码 → 完成提示。

【台词】
- “备份文件是加密 JSON：先序列化分组与条目，再 PBKDF2 派生 key，加密后写入 `.tbxpm` 文件；导入时解密并恢复数据结构。”

## 10) 7:30–8:20 可选加分：Web 助手（内置 WebView 演示闭环）
【操作】
1) 点击顶部 `Web 助手` → 点 `演示页`。
2) 在密码框右侧点击蓝色 `钥` 按钮 → 选择 `demo.tbx` 对应条目 → 自动填充账号/密码。
3) 点页面 `提交登录`，观察弹出的“保存/更新/忽略”提示（任选一种演示）。

【台词】
- “这里演示的是‘内置 WebView 的集成’，完成‘识别密码框 → 一键填充 → 提交后提示保存/更新’闭环。”
- “实现基于 `Qt WebEngine + QWebChannel`：注入脚本在 DOM 中定位 `input[type=password]` 并插入‘钥’按钮；点击后通过 WebChannel 调用 Qt 侧 bridge，弹出选择项并回填。”
- “注意：这不是系统浏览器插件；要覆盖 Chrome/Edge 的真实网页填充，需要额外做浏览器扩展或本地通信桥接。本项目把它作为可选加分模块，核心验收不依赖它。”
